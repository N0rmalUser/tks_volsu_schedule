# Работа с ботом для самых маленьких

## Как запустить бота
* Установить все зависимости из `requirements.txt`
* Запустить файл main.py

## Где находятся настройки бота
Файл с настройками находится в папке `data/config.py`

## Как получить файл с расписаниями
* Расписания колледжа (`xlsx`) и университета (`docs`) засунуть в `/подготовка/расписания/  `
* Затем запустить файл `university_schedule.py` и `college_schedule.py  `
* После запустить json_combiner.py  
* Переместить полученный файл в /data/ (есл в config.py не указан другой путь)

## Как отправлять сообщения пользователям бота
В `config.py` есть переменная `GROUP_CHAT_ID`, в которой указывается id чата,
в котором создаются топики пользователей, нажавших на кнопку "Начать" в боте.  
Написав сообщение в этот чат, оно будет отправлено этому пользователю.  
Если отправить сообщение в "General", сообщение получат все пользователи бота.

## Какие команды поддерживает бот
### Обычные юзвери тогут использовать только:
* `/start` - Начать работу с ботом
* `/admin` - Написать админу (в админском чате появится сообщение с просьбой о помощи
и автоматом врубится слежение за действиями пользователя)

### Админы могут использовать те же команды, что и юзвери, а также:
* `/schedule_update` - Обновляет переменную расписания без перезапуска бота
* `/track start` - Включить слежение за действиями пользователя (если написан в топике,
то только за этим юзером, если в "General", то за всеми)  
* `/track stop` - Выключить слежение за действиями пользователя. Те же правила, что и при start  
* `/track status` - Проверить, включено ли слежение за действиями пользователя. Если написать в General, показывает список пользователей, за которыми включено слежение
* `/info` - Присылает всю ключевую инфу пользлователя из бд (не работает в "General")  
__Если команда написана в группе, она не перешлётся пользователям__

При желании можно переименовывать команды в `config.py` (не добавлена)  
Можно переименовывать топики и изменять их иконки, это ни как не повлияет на работу бота
